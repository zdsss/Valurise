# Valurise 用户测试指南

## 测试目标

在Week 1验证阶段，我们需要邀请5-10位潜在用户测试Valurise原型，收集真实反馈以决定是否继续开发Web版本。

---

## 如何选择测试用户

### 理想的测试用户画像

**优先邀请**：
1. **正在求职的朋友**
   - 最近在找工作或准备跳槽
   - 需要更新简历
   - 对职业发展有困惑

2. **职业转型者**
   - 工程师想转产品/运营
   - 传统行业转互联网
   - 个人贡献者想晋升管理层

3. **应届毕业生**
   - 2024/2025届毕业生
   - 正在准备秋招/春招
   - 不知道如何写简历

4. **资深专业人士**
   - 5-10年工作经验
   - 寻求更高职位
   - 需要重新定位自己

**避免邀请**：
- 完全不需要求职的人（测试动机不足）
- 对AI/科技产品完全不感兴趣的人
- 没有时间认真测试的人

## 测试流程

### 准备阶段（测试前）

1. **发送邀请**
   - 使用上面的话术模板
   - 说明测试目的和时间
   - 强调保密性（如果需要）

2. **确认参与**
   - 确定测试时间
   - 发送准备材料清单
   - 提醒提前准备职业信息

3. **发送准备清单**
```
请提前准备以下信息：
□ 工作经历（公司、职位、时间、主要成就）
□ 教育背景
□ 技能列表
□ 目标岗位和行业
□ 职业转型原因（如果有）

格式不限，可以是：
- 现有简历
- 文字描述
- 语音转文字
- 随意的笔记
```

---

### 测试阶段（测试中）

#### 方式1: 远程测试（推荐）

1. **收集用户输入**
   - 让用户发送职业信息
   - 确认目标岗位

2. **运行Valurise**
   ```bash
   cd /Users/monet/Program/Repo/Valurise/prototype
   python main.py
   # 或者修改main.py使用用户的输入
   ```

3. **发送结果**
   - 发送生成的JSON文件
   - 或者整理成易读的格式发送

4. **收集反馈**
   - 发送反馈问卷
   - 可以通过视频通话讨论

#### 方式2: 面对面测试

1. **准备环境**
   - 打开终端和Valurise
   - 准备记录工具

2. **输入信息**
   - 让用户口述或输入职业信息
   - 实时运行系统

3. **展示结果**
   - 一起查看输出
   - 观察用户反应
   - 记录即时反馈

4. **深度访谈**
   - 询问具体感受
   - 探讨改进建议
   - 了解付费意愿

---

### 反馈收集（测试后）

1. **发送问卷**
   - 使用 `user_feedback_form.md`
   - 可以转成Google Form或问卷星

2. **关键问题**（如果用户时间有限，至少问这些）：
   - 整体满意度（1-5分）
   - 最喜欢什么？最需要改进什么？
   - 是否愿意付费？愿意支付多少？
   - 推荐意愿（0-10分）

3. **记录到测试日志**
   - 更新 `test_log.md`
   - 记录关键洞察
   - 标注重要引用

## 测试注意事项

### Do's ✅

1. **保持中立**
   - 不要引导用户给好评
   - 鼓励真实反馈
   - 接受负面意见

2. **深入挖掘**
   - 问"为什么"
   - 了解具体场景
   - 探索真实需求

3. **记录细节**
   - 用户的原话
   - 表情和反应
   - 使用场景

4. **感谢参与**
   - 真诚感谢
   - 兑现承诺（报告、折扣等）
   - 保持联系

### Don'ts ❌

1. **不要辩解**
   - 用户说不好，不要解释"这是因为..."
   - 记录问题，不要争论

2. **不要过度推销**
   - 这是测试，不是销售
   - 不要强迫用户说愿意付费

3. **不要忽略负面反馈**
   - 负面反馈最有价值
   - 认真对待每个问题

4. **不要泄露隐私**
   - 保护用户信息
   - 不要分享给第三方

## 常见问题

### Q: 如果用户不愿意提供真实信息怎么办？
A: 可以使用化名和模糊化的公司名称，重点是职业经历的结构和成就。

### Q: 如果用户说"我不知道自己的价值"怎么办？
A: 这正是Valurise要解决的问题！让他们提供基本信息，看AI能发现什么。

### Q: 如果用户对结果不满意怎么办？
A: 这是宝贵的反馈！深入了解哪里不满意，为什么，期望是什么。

### Q: 如何处理用户的隐私信息？
A:
- 明确告知数据用途
- 不保存敏感信息
- 测试后可以删除数据
- 报告中可以匿名化
